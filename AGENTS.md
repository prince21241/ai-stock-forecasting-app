# Repository rules

- Keep API routes thin and place business logic in services.
- Put database access in repositories.
- Use type annotations and async APIs where appropriate.
- Never commit secrets or log credentials.
- Add tests for behavior changes and run tests and linters before completion.
- Preserve the modular architecture.
- Do not claim unfinished forecasting or AI-agent features exist.
- Keep future phases separate from Phase 1.
- Update the README whenever setup or behavior changes.
